Managing Execution

An OS juggles countless tasks. This is achieved through processes and threads, the fundamental units of work.

Process vs. Thread

A process is an isolated program (like a browser), while threads are concurrent tasks within it (like tabs and spell-check).

Process (Heavyweight)

Has its own private memory. A crash is isolated. Slower to create and switch.

Thread (Lightweight)

Shares memory with other threads in the same process. Faster to create and switch.

CPU Scheduling Algorithms

The OS scheduler decides which process runs next. Different algorithms have different trade-offs in fairness and efficiency.

The chart above visualizes how different scheduling algorithms would execute the same set of jobs. Notice how Shortest Job First (SJF) minimizes overall wait time but risks starving longer jobs, while Round Robin (RR) provides more fairness by giving each job a turn.

Memory Management

The OS acts as a real estate agent for RAM, allocating space to programs while creating the illusion of infinite memory.

Paging: The Great Equalizer

Memory is divided into fixed-size "frames," and programs into same-sized "pages." This eliminates unusable gaps (external fragmentation).

Segmentation: The Logical View

Memory is divided into logical, variable-sized "segments" (like code, data, stack), but this can lead to fragmentation.

Virtual Memory: The Ultimate Illusion

By using disk space as an extension of RAM, the OS lets you run programs far larger than your physical memory. Only the active parts of a program are kept in RAM.

File System Management

The OS organizes data on disks, providing the familiar structure of files and folders.

File Allocation Methods

How does the OS store file blocks on a disk? Each method has pros and cons.

Contiguous

Fast access, but suffers from fragmentation.

Linked

Flexible, but slow for random access.

Indexed

Good balance, but has overhead for the index block.

Modern File Systems

Different OSes use different file systems, each optimized for specific goals like reliability, performance, or compatibility.
